# TSH AI インターフェース: 物理シミュレーション解析 ＆ パラメータ制御 API
## 実装マニュアル (v2.0)

---

### 0. ファイル構成と役割 (File Inventory)

本ディレクトリは、AI が物理シミュレーションデータを読み取り、シミュレーション設定（パラメータ）を動的に変更するための Python インターフェースを提供します。

- **tsh_ai_api.py** (シミュレーション解析 ＆ 制御 API)
  - 粒子の状態（位置、速度、エネルギー等）を取得し、解析や外部学習モデルへの受け渡しを行うための API クラスです。
- **materials.json** (マテリアル設定ファイル)
  - 粒子の種類ごとの物理定数（相互作用の強さなど）を定義した設定ファイルです。API を通じて書き換えることで、シミュレーションの挙動を即座に変更できます。

---

### 1. 概要

本インターフェースは、物理シミュレーションを AI 学習の「環境」として利用するための標準的なデータ入出力（I/O）レイヤーです。

1. **データ取得 (Data Acquisition)**: シミュレーション内の物理量（有効質量、合計エネルギー、場密度など）を NumPy 配列等として取得し、AI の入力データとして利用可能にします。
2. **パラメータ制御 (Parameter Control)**: マテリアル設定（$\alpha, \beta$ 等）を最適化することで、特定の物理現象（粒子の集約や安定化）を誘発するような法則の探索を支援します。

---

### 2. TSHAI_API の主要機能

#### ■ パラメータの動的更新 (Agency)
- **`edit_material()`**: 
  `materials.json` 内の物理定数を書き換えます。シミュレーター側がこのファイルをリロードすることで、実行中の宇宙の法則が変化します。

#### ■ 物理状態の数値評価 (Evaluation)
- **`evaluate_phase_topology()`**: 
  場の密度分布から「どれだけ粒子が密集しているか（core_density）」や「トポロジーの複雑さ（structural_entropy）」を数値化します。
- **`evaluate_irreversibility()`**: 
  系の収縮度合いや不可逆性をスコア化し、AI の学習における報酬（Reward）として利用できます。

#### ■ 学習用データのエクスポート (Data Export)
- **`export_observables()`**: 
  シミュレーションの観測データを `.npy` 形式で保存します。これにより、PyTorch や TensorFlow などの外部フレームワークで、物理現象を教師あり学習させることが容易になります。

---

### 3. 標準的な利用ワークフロー

1. **観測**: API を使用してシミュレーションの状態を取得し、評価スコアを算出。
2. **推論**: スコアに基づき、外部 AI モデルがより良い物理パラメータを推論。
3. **反映**: API を使用して `materials.json` を更新。
4. **検証**: 更新された環境でシミュレーションを続行し、結果を確認。
